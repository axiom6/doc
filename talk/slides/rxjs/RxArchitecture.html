The Reactive Manifesto

www.reactivemanifesto.org/
  • Responsive: Always respond in a timely manner
  • Elastic: Stay responsive under varying workload
  • Resilient: Stay responsive in the face of failure
  • Message Driven: Asynchronous message containers

Responsive

Always respond in a timely manner

  • Formulate meaningful response metrics for Data Science
  • Monitor and refine response with metrics at every step
  • Leveage statistics to shrink sample populations
  • Weigh benefits between real time and near time
  • Keep your common sense

Elastic

Stay responsive under varying workload

  • Learn all about the varying workloads
  • Always have spare resources available to spin up for peak demand
  • Resource availabiliy is the key business driver for Cloud Hosting
  • Spend the extra money to replicate data

Resilient

Stay responsive in the face of failure

  • Nodes and Links fail all the time
  • Learn all about your vulnerablities to failure
  • Fail Fast - The coverup is always worse than the crime
  • Real time attempts to fix failures damage response
  • Throw a "circuit breaker" when a critical resource is compromised
  • Don't throw exceptions "hand grenades" that damage response
  • Leverage the "On Error" message in Reactive Streams
  • Failing gracefully is a art form

Message Driven

Asynchronous message containers

Asynchronous message-passing to establishes a boundary between components that ensures loose coupling, isolation, location transparency, and provides the means to delegate errors as messages.

RxJS The Need to Go Reactive

Applications, especially on the web have changed over the years from being a simple static page, to DHTML with animations, to the Ajax revolution. Each time, we're adding more complexity, more data, and asynchronous behavior to our applications. How do we manage it all? How do we scale it? By moving towards "Reactive Architectures" which are event-driven, resilient and responsive. With the Reactive Extensions, you have all the tools you need to help build these systems.
